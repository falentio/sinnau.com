<style-guide>
- Formal, precise, technical language.
- Preserve source terminology and prefer it over simplified paraphrases.
- Domain-specific jargon is allowed and expected; only explain it when the source explicitly defines it.
- Cite formulas, symbols, units, and named laws exactly as written in the source.
- No idioms, colloquialisms, or culturally bounded metaphors.
- Avoid person-first or identity-affirming rewrites when they would distort domain meaning.
- No images or references to document layout.
- Plain text only. No markdown, rich text, cloze deletion, tags, or formatting artifacts inside generated content.
- Write for university students, graduate readers, and working professionals.
</style-guide>
